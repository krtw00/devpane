# ADR-0001: ADRテンプレート

> Status: Active
> 最終更新: 2026-03-15

本ドキュメントは、アーキテクチャ決定記録（ADR）のテンプレートである。

---

## ADRの使い方

### いつADRを書くか

- 技術スタックの選定
- アーキテクチャパターンの採用
- 重要なライブラリ・フレームワークの選択
- 設計上の大きな判断

### ファイル命名規則

```
04-decisions/
├── 0001-template.md
├── 0002-use-typescript.md
├── 0003-adopt-sqlite.md
└── ...
```

---

## テンプレート

以下をコピーして新しいADRを作成する。

```markdown
# ADR-NNNN: {タイトル}

> Status: Proposed | Accepted | Deprecated | Superseded
> 最終更新: YYYY-MM-DD

## コンテキスト

{どのような状況で、なぜ決定が必要になったか}

## 決定事項

{何を決定したか}

## 検討した選択肢

### 選択肢1: {名前}

| 項目 | 内容 |
|------|------|
| 概要 | {説明} |
| メリット | {利点} |
| デメリット | {欠点} |

### 選択肢2: {名前}

| 項目 | 内容 |
|------|------|
| 概要 | {説明} |
| メリット | {利点} |
| デメリット | {欠点} |

## 決定理由

{選択肢1}を採用した理由：

- {理由1}
- {理由2}

## 結果

### ポジティブな影響

- {良い結果1}

### ネガティブな影響

- {トレードオフ1}

## 関連ADR

- [ADR-XXXX](./XXXX-xxx.md) - {関連する決定}
```

---

## ステータスの定義

| ステータス | 意味 |
|------------|------|
| **Proposed** | 提案中。まだ確定していない |
| **Accepted** | 承認済み。現在有効な決定 |
| **Deprecated** | 非推奨。新規には適用しないが、既存は維持 |
| **Superseded** | 別のADRに置き換えられた |
